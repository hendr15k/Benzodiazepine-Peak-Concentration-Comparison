# Contributing Guidelines

Thank you for your interest in contributing to our project! From commenting on and triaging issues, to reviewing and sending Pull Requests, all contributions are welcome. We aim to make the process as straightforward as possible and are excited to help new contributors.

## Getting Started 

1. Fork the repository on GitHub

2. Clone the project to your machine

```bash
git clone https://github.com/hendr15k/benzodiazepine_concentration.git
```

3. Navigate to the project folder and create a new branch where you'll make your changes

```bash
cd benzodiazepine_concentration
git checkout -b name_for_new_branch
```

4. Make your changes in the new branch

## Making a Pull Request

Once you've pushed a change to your forked repo, you can then submit a pull request. To do this:

1. Navigate to your forked repo on GitHub

2. Click on 'New Pull Request' 

3. You should be able to see your changes and are able to write a brief description of what the changes are

4. Submit the Pull Request

## Reporting Bugs and Issues

If you've encountered a bug or have a feature request, please create an [issue](https://github.com/hendr15k/benzodiazepine_concentration/issues) with a detailed description of what you experienced or what you'd like to see added.

## Code Review

Pull Requests will be reviewed by the repository owners. Feedback will be provided if any changes are necessary.

## Community

You can help us in different ways:

- Report bugs and answer questions on bug reports.

- Review the source code changes.

- Help us in improving our docs.

We appreciate all contributions, and we are excited to see what the community can provide!
